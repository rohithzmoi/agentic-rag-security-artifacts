# Artifact Package: SoK: Authorization in Multi-Agent RAG Systems

This artifact package supports reproducibility of the systematization claims in the paper.

## Contents

```
artifacts/
├── README.md                 # This file
├── data/
│   ├── defense_families.csv  # Table 1 source data (D1-D6)
│   ├── deployment_patterns.csv # Table 2 source data (P1-P6)
│   └── failure_modes.csv     # Failure modes F1-F5
├── scripts/
│   ├── generate_tables.py    # Regenerates tables + statistics
│   └── requirements.txt      # Python dependencies
└── outputs/                  # Generated by scripts (not included)
```

## Quick Start

```bash
# Install dependencies
cd /
pip install -r scripts/requirements.txt

# Generate tables and statistics
make tables
```

Alternatively, run the Python script directly:

```bash
cd scripts && python generate_tables.py
```

## Expected Outputs

Running `generate_tables.py` produces:

1. `outputs/table1_defense_families.md` - Defense family guarantee matrix (matches paper Table 1)
2. `outputs/table2_deployment_patterns.md` - Deployment patterns matrix (matches paper Table 2)
3. `outputs/summary_statistics.txt` - Statistics supporting claims O1-O5

## Data Sources

All data is derived from the paper's systematization:

| File | Paper Section | Description |
|------|---------------|-------------|
| `defense_families.csv` | Section 7, Table 1 | Defense families D1-D6 with AFR/Auth/Deleg guarantees |
| `deployment_patterns.csv` | Section 8, Table 2 | Representative deployment patterns P1-P6 |
| `failure_modes.csv` | Section 6 | Authorization failure modes F1-F5 |

## Verification

The statistics output verifies the paper's key observations:

- **O1**: D3 and D4 cannot satisfy AFR (afr_support = No)
- **O2**: D1 and D2 have conditional AFR (depends on tag freshness)
- **O3**: D1-D4 have no delegation correctness guarantee
- **O4**: Only AFR-oriented (D6) achieves full AFR support
- **O5**: No pattern simultaneously achieves AFR + Delegation + State Isolation with full guarantees
